How PayprAPI Works

The 402 protocol: demand, pay, receive.

01

Request Call

Your agent makes a standard REST call to our endpoint without any prior auth.

02

402 Required

We return a 402 Payment Required status with a specific Algorand transaction hash.

03

Instant Settlement

Agent sends the tiny micro-payment. Verified instantly via blockchain finality.

04

Result Delivery

API content is unlocked and delivered with zero latency overhead.

    # Get the invoice details from headers
